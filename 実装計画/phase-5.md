## 1. スケール不変性の原理

本システムの階層構造は、物語のスケールに関わらず4層で表現される。

```
層1: 環境層     — キャラクターを取り巻く「変えられない条件」
層2: 所属層     — キャラクターが属する「集合的な意志」
層3: 対人層     — キャラクター同士の関係性と相互作用
層4: 内面層     — キャラクターの内面、秘密、動機
```

各層に何が配置されるかはスケールによって変化する:

| 層 | 大陸規模 | 都市規模 | 家庭規模 | 個人内面規模 |
|---|---|---|---|---|
| 環境層 | 気候・地殻変動 | 都市の景観・経済 | 家の間取り・生活音 | 体調・感覚・時間感覚 |
| 所属層 | 国家・帝国 | 行政区・企業 | 家族・同居人 | 人格の側面・内なる声 |
| 対人層 | 外交・同盟 | 組織間関係 | 家族内の関係 | 自己内対話 |
| 内面層 | 国民感情・時代精神 | 世論・文化 | 家庭の空気 | 感情・記憶・無意識 |

## 2. スケールイン — 焦点を絞る操作

### 深度レベル

```
レベル1（浅い）:
  1段階焦点を絞る。数シーンで戻る一時的な集中
  例: 戦争の中の「一夜の野営シーン」
  操作: 一部アクターの一時休眠。新規アクター追加なし
  戻るコスト: 低（数シーン分のサマリ）

レベル2（深い）:
  2段階以上焦点を絞る。サイクルの一部を占める
  例: 政治劇の中で「主人公の少年時代の回想」を1サイクル挿入
  操作: 大半のアクターを休眠。新規アクター追加あり
  戻るコスト: 中（1サイクル分のサマリ + 新規アクターの処理）

レベル3（完全）:
  物語のスケール自体が恒久的に変化する
  例: 戦記から、戦後の一つの家族の物語へ完全に移行
  操作: アクター構成を根本的に再編成
  戻るコスト: 戻らない（不可逆）またはGitロールバック
```

### 実行手順

```
1. EDT が「スケールイン宣言」をコミット
2. スケールイン対象外のアクターを休眠に移行
3. 焦点範囲内のアクターの解像度を上げる
   - CHR-* の更新頻度: シーン単位 → 会話ターン単位
   - 環境描写: マクロ → ミクロ
   - 所属アクター: 組織の公式動向 → 組織内の人間関係
4. 新規の微細アクターを必要に応じて追加
```

## 3. スケールアウト — 焦点を広げる操作

### 速度パターン

```
急激なスケールアウト:
  1-2シーンで世界が一気に広がる
  効果: 衝撃、世界観の転覆
  リスク: 読者の混乱、既存キャラクターの埋没

漸進的スケールアウト:
  数シーン〜数サイクルかけて徐々に世界が広がる
  効果: 自然な拡張、読者の理解が追いつく

パルス的スケールアウト:
  特定のシーンでだけ大きな視野が開け、すぐに元のスケールに戻る
  効果: 「今いる場所の外に大きな世界がある」という暗示
```

### 実行手順

```
1. EDT が「スケールアウト宣言」をコミット
2. 新たな上位アクターの追加（NAT-*, ORG-*）
3. 既存アクターの「所属」の再定義
4. 情報可視性の拡張（段階的情報開示を適用）
```

## 5. スピンアウト — 物語からの分岐独立

### 5.1 スピンアウトの5種類

```
■ キャラクタースピンアウト
  脇役や敵役を主人公とした別の物語
  抽出: CHR-X のハンドアウト, recall, 関係性マップ, 所属組織

■ 場所スピンアウト
  元の物語に登場した場所を舞台にした別のキャラクターの物語
  抽出: HO-PUB の場所関連部分, 環境・所属アクターのデータ

■ 時間軸スピンアウト
  元の物語の過去または未来を舞台にした物語
  過去: 回想の断片を完全に描く
  未来: TRK-CSQ の未発現帰結が初期条件に

■ 組織スピンアウト
  脇役的だった組織を中心にした物語
  例: 冒険パーティの背景にあった「商人ギルド」の経済ドラマ

■ 廃棄世界線スピンアウト
  Git で廃棄されたブランチ（不採用展開）を独立した物語として蘇生
  最も Git の恩恵を受ける形態
```

### 5.2 情報可視性ルール

```
■ 絶対に持ち込むもの
  - HO-PUB の関連部分（世界設定の一貫性保証）
  - スピンアウト対象の公開情報

■ 選択的に持ち込むもの（EDT + HMN の判断）
  - 非公開ハンドアウト（公開前提にするか秘密を維持するか）
  - 確定済みシーン（バックストーリーとして参照可能にするか）
  - 関係性マップ（引き継ぐかリセットするか）

■ 絶対に持ち込まないもの
  - HO-GM（新しい物語には新しい GM ハンドアウト）
  - TRK-FSH の全体（伏線構造は別物語の制約にならない）
  - EVL-*（新しい読者体験）
  - EDT 判断記録（新しい EDT の独自判断）
```

### 5.3 元の物語との整合性: 3モード

| モード | 制約 | 実装 |
|---|---|---|
| **正史連結** | 元の確定事実に矛盾禁止。共有 HO-PUB の同期 | サブモジュール化。相互フック |
| **準正史** | 消極的整合性（矛盾しないが影響なし） | HO-PUB を読み取り専用参照。一方向依存 |
| **独立** | 制約なし。パラレルワールド | fork 後の完全独立リポジトリ |

### 5.4 Git 操作

```bash
# キャラクタースピンアウト（正史連結）
applicot spinoff character chr_x --mode canon-linked
  # → 元リポジトリから CHR-X 関連データを抽出
  # → 不要なキャラクターデータを削除
  # → spinoff-origin/chrX-story タグ付け
  # → 元リポジトリをリモート登録（整合性チェック用）

# 廃棄世界線スピンアウト
applicot spinoff branch branch-alliance-abandoned --mode independent
  # → 廃棄ブランチから新リポジトリを作成
  # → ブランチの最終状態を起点として新物語を開始

# 場所スピンアウト
applicot spinoff location "辺境の村" --mode semi-canon
```

## 6. スケール変化時のアクター稼働ルール

```
■ スケールイン時
  レベル1: 上位アクター → 休眠（一時的）
  レベル2: 上位アクター → 休眠（中期的）、新規微細アクター追加
  レベル3: 上位アクター → 永久休眠/廃棄、全面再編

■ スケールアウト時
  漸進的: 新規上位アクター → 待機で追加、段階的に活性化
  急激: 新規上位アクター → 即座に活性。読者混乱度チェックが必要

■ スピンアウト時
  新リポジトリのアクター群は元と完全独立した稼働スケジュール。
  正史連結でも稼働タイミングの同期は行わない。
  整合性は状態データのクロスチェック（非同期）で担保。
```

## 7. 組み合わせパターン

### 呼吸パターン（スケールの伸縮）

```
サイクル1: 大規模（戦争の開始）
  ↓ スケールイン
サイクル2: 小規模（一兵士の体験）
  ↓ スケールアウト
サイクル3: 大規模（戦局の転換）
  ↓ スケールイン
サイクル4: 小規模（帰還した兵士の家庭）
```

### 入れ子パターン（フラクタル構造）

大きな起承転結の中に小さな起承転結が入れ子になる。
外側と内側で四象限評価を二重に追跡。

### 望遠鏡パターン（段階的スケールアウト）

サイクルごとに不可逆的にスケールが拡大していく。
